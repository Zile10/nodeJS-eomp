API Routes:

Home Route

# Request Route URL Description
1 GET / Root Route. Responds with this page.

User Routes

# Request Route URL Description
1 GET /users Fetches all users from DB & responds with results if the requesting user is an admin.
2 POST /users Inserts new users in DB with given values. (Used to register new user)
3 POST /users/login Checks DB to verify user email and password, and signs a JWT, if valid user.
4 GET /users/:id Fetches specific user & responds with the results if the requesting user is an admin.
5 PATCH /users/:id Finds and updates specific user with given values, if the requesting user is an admin.
6 DELETE /users/:id Finds and deletes specific user, if the requesting user is an admin.

Product Routes

# Request Route URL Description
1 GET /products Fetches all products from database and responds with the results.
2 POST /products Inserts new product in DB with given values, if the requesting user is an admin.
3 GET /products/:id Fetches specific product & responds with the results if user is logged in.
4 PATCH /products/:id Finds and updates specific product with given values, if the requesting user is an admin.
5 DELETE /products/:id Finds and deletes specific product, if the requesting user is an admin.
